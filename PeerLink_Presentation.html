PeerLink

02 / 11
Motivation

The Problem

🌐

Internet Dependency

All popular messaging apps rely on cloud servers and active internet connectivity to exchange messages.

🚫

Dead Zones

Remote areas, disaster zones, underground venues, and events often have no cellular signal or Wi-Fi infrastructure.

🔒

Privacy & Surveillance

Server-based messaging exposes messages to third-party cloud providers, creating privacy and security concerns.

💡 The Need: A simple, infrastructure-free communication method that works with just a small device and smartphones.
03 / 11
Solution

Introducing PeerLink

PeerLink transforms an ESP32 microcontroller into a portable Wi-Fi relay hub. Nearby Android phones connect to it and can chat with each other — completely offline, no SIM card, no internet, no server.

  • Works anywhere — even in remote areas with zero connectivity
  • No registration, no accounts, no cloud dependency
  • ESP32 costs under ₹400 and runs on a phone charger
  • End-to-end local — messages never leave the local network
📡

ESP32 Acts As:


• Wi-Fi Access Point (hotspot)
• UDP packet relay / router
• Client registry & tracker
• Presence broadcaster

Hardware Layer

ESP32 Relay Firmware

  • Wi-Fi AP Mode — Broadcasts SSID PeerLink_XXXX (unique MAC suffix)
  • UDP Server on port 8888 — Listens and routes all packets
  • Client Registry — Tracks all connected device IDs, IPs, names
  • Presence Broadcast — Announces relay every 2 seconds via UDP broadcast
  • Client Timeout — Removes inactive devices after 30 seconds
  • LED Indicator — GPIO 2 blinks on packet activity

06 / 11
Android Layer

Android Application

🔌

EspConnector

Uses Android's WifiNetworkSpecifier API to automatically request and connect to PeerLink_* hotspot networks via a Kotlin callbackFlow, binding the process to the specific network.

📦

UdpChat

Manages a DatagramSocket on port 8888 for sending and receiving JSON-encoded UdpPacket objects. Uses Kotlin Coroutines (IO Dispatcher) and MutableSharedFlow for reactive message streaming.

⚙️

MeshService

Android Foreground Service that stays alive in the background. Orchestrates EspConnector, UdpChat, and Room DB. Exposes serviceState as a StateFlow to the UI layer.

🗄️

Room Database

Persists all incoming and outgoing MessageEntity records locally. UI observes messages via a Flow<List<MessageEntity>>, enabling real-time automatic updates.
